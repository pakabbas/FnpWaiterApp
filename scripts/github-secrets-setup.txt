GitHub CI — secrets and variables (APK + Play AAB)
Repo: https://github.com/pakabbas/FnpWaiterApp

=== 1. Open settings ===
Repository → Settings → Secrets and variables → Actions

=== 2. Repository secrets (tab: Secrets) — New repository secret ===
Create these four (same names, case-sensitive):

  RELEASE_KEYSTORE
    Value: Base64 of your .jks file (PowerShell):
      [Convert]::ToBase64String([IO.File]::ReadAllBytes("D:\src\waiterApp\android\app\foodnpals-release-key.jks"))
    Paste the ONE long line (no quotes).

  STORE_PASSWORD
    Value: from android\key.properties → the text after storePassword=

  KEY_PASSWORD
    Value: from android\key.properties → the text after keyPassword=

  KEY_ALIAS
    Value: e.g. key0 (from keyAlias= in key.properties)

=== 3. Repository variable (tab: Variables) — for Play Store AAB only ===
  Name:  PLAY_STORE_VERSION_OFFSET  (OPTIONAL, default base 1000000)

  CI computes:
    versionCode = OFFSET + (workflow run number × 1000) + run attempt

  Run attempt is 1 on first try and goes up when you "Re-run jobs", so re-runs never reuse the same
  versionCode as a failed first try.

  Raise OFFSET only if you need a higher floor (e.g. to stay above an old manual upload).

=== 4. Enable Actions ===
Settings → Actions → General → allow Actions (default).

=== 5. One workflow per push ===
Workflow file: .github/workflows/fnp_android_release.yml
Workflow name in UI: "FnP Android Release"
  - One run on each push to main/master; two parallel jobs: Signed release APK + Play Store AAB
  - Artifacts: release-apk-v<versionCode> (app-release.apk), release-aab-v<versionCode> (app-release.aab)

You can also run it manually: Actions → FnP Android Release → Run workflow.
