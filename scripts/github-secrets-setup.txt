GitHub CI — secrets and variables (APK + Play AAB)
Repo: https://github.com/pakabbas/FnpWaiterApp

=== 1. Open settings ===
Repository → Settings → Secrets and variables → Actions

=== 2. Repository secrets (tab: Secrets) — New repository secret ===
Create these four (same names, case-sensitive):

  RELEASE_KEYSTORE
    Value: Base64 of your .jks file (PowerShell):
      [Convert]::ToBase64String([IO.File]::ReadAllBytes("D:\src\waiterApp\android\app\foodnpals-release-key.jks"))
    Paste the ONE long line (no quotes).

  STORE_PASSWORD
    Value: from android\key.properties → the text after storePassword=

  KEY_PASSWORD
    Value: from android\key.properties → the text after keyPassword=

  KEY_ALIAS
    Value: e.g. key0 (from keyAlias= in key.properties)

=== 3. Repository variable (tab: Variables) — for Play Store AAB only ===
  Name:  PLAY_STORE_VERSION_OFFSET  (OPTIONAL)

  If you do NOT set it:
    versionCode = 1000000 + workflow run number (e.g. 1000001, 1000002) — avoids "version 3 already used" when
    you once uploaded low numbers manually.

  If you WANT small sequential codes (4, 5, 6…):
    Set the value to the HIGHEST versionCode already shown in Play Console (Releases / App bundle explorer).
    Formula: versionCode = OFFSET + run number. Example: max on Play was 3 → set OFFSET=3 → next builds 4,5,6…

  If Play says "version code X already used": raise OFFSET to at least X (or leave unset to use the 1000000+ default).

=== 4. Enable Actions ===
Settings → Actions → General → allow Actions (default).

=== 5. One workflow per push ===
Workflow name: "Android Release (APK + Play Bundle)"
  - One run on each push to main; two parallel jobs: Release APK + Play Store AAB
  - Artifacts: release-apk (app-release.apk), release-aab-vNNN (app-release.aab)

You can also run it manually: Actions → Android Release (APK + Play Bundle) → Run workflow.
